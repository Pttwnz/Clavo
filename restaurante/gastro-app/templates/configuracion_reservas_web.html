Dos conceptos distintos:
  • Franjas web (esta página): definen en qué horas se puede reservar online y el % del aforo reservable en cada tramo (solo cuentan reservas con origen «web» en pendiente / confirmada / llegó).
  • Rangos mañana / tarde / noche en Configuración empresa: sirven sobre todo para horarios de personal y vistas del módulo de horarios; no sustituyen las franjas web.
  • Reservas manuales desde el panel no están bloqueadas por estas franjas (sí por mesa y solapes); conviene que los horarios coincidan con lo que ofrecéis en la web.

Franjas horarias para la web

Aquí defines cuándo el cliente puede elegir hora en la web y cuántas personas como máximo pueden reservar online en ese tramo (sin contar reservas del panel ni walk-in).

Cómo funciona (léelo una vez)
  1. Día: cada fila es un tramo de un día de la semana (lunes a domingo, norma ISO 1–7). El mismo servicio repetido cada día requiere siete filas o el botón «A toda la semana».
  2. Inicio / Fin: ventana horaria local. Dentro de ella, la web genera huecos cada intervalo de minutos (arriba). Ej.: 13:00–16:00 con intervalo 30 → 13:00, 13:30, 14:00…
  3. Etiqueta: texto opcional (ej. «Comida») que ve el cliente al elegir hora.
  4. % web: porcentaje del aforo total del salón ({{ aforo }} pax) reservable online en ese tramo. Vacío = se usa «% web por defecto» de arriba. El cupo se reparte entre reservas web en pendiente / confirmada / llegó.
  5. Cupo (~): aproximación aforo × % ÷ 100 (el motor puede reservar 1 plaza mínima si el % es bajo).
  6. Activa: «No» desactiva la fila sin borrarla (útil para guardar plantillas).

Al guardar se ignoran filas sin día o sin hora de inicio/fin. No puede haber dos franjas activas solapadas el mismo día. Tras guardar, revisa la vista previa de huecos más arriba.

Puedes tener muchas filas; solo cuentan las que tengan día y horas válidas.
